## transition semantics v1

### 計画

- runtime 実装に分散している transition の `from` / `to` / `role` / `guard` / `input` semantics を `docs/transition-semantics-v1.md` に固定する
- failure diagnostics と runtime 関数の対応を表にして、consumer が実装根拠を追えるようにする
- roadmap の issue 参照を closed path に合わせ、issue を close する

### やったこと

- `docs/transition-semantics-v1.md` を追加し、`apply_transition` と `list_transitions` の評価順、診断蓄積、read-only / mutating 境界を文書化した
- `STATE_MISMATCH`、`ROLE_DENIED`、`GUARD_REJECTED`、`INVALID_TRANSITION_INPUT` などの failure code と意味を整理した
- rule と view は authorization ではなく projection / facts であり、mutation enforcement は runtime transition function が担うことを明文化した
- README と roadmap から transition semantics 文書を参照し、issue を `issues/closed/` に移動した

### 学んだこと

- transition は UI の action 表示だけではなく、state field、actor role、guard、record validation、input validation が同じ順序で評価される runtime contract として扱う必要がある
- transition-local input は record に保存されないが、required/default/type validation の対象になるため、durable host は audit や transport payload と record mutation を分けて扱う方が安全になる
