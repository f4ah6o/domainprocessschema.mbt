# 2026-05-10

## App Server schema tools

今日は runtime session store の上に App Server tool 境界を追加する。対象は
`app_server_schema_tools.mbt` と公開型を置く `domainprocessschema_mbt.mbt`、確認用の
`app_server_schema_tools_test.mbt` に絞る。

read-only、mutating、approval-required を型で分け、HTTP や永続化 adapter は入れない。
approval-required は実行結果ではなく approval proposal、つまり client が承認 UI に渡す
ための提案データを返す形にする。

## App Server schema/runtime session contract closeout

`issues/2026-05-09T093100-architecture-define-codex-app-server-schema-and-runtime-session.md`
は後続の session store / tool / artifact 実装でほぼ満たされていたので、今日は不足分だけを
埋めた。具体的には diagnostic を `code`、`severity`、`target`、`message`、`hint`、
`context` の形に揃えた。diagnostic は compile や runtime の失敗を client が機械的に扱う
ための構造化エラーで、ただの表示用文字列ではなく、どこで何が起きたかを stable JSON として
返す contract になる。

session snapshot は `source`、`selection`、`actor`、`record`、`compile`、`runtime`、
`graph`、`locale`、`lastAction` を持つ。read-only tool は approval 不要、transition と
reset は approval-required という境界も `app_server_schema_tools.mbt` の catalog で確認できる。
runtime core には HTTP / UI / persistence adapter を入れず、App Server 側がこの library
contract を消費する形を維持した。
