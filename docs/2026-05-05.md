## 学んだこと

- **stacked PR** は、前の PR の差分を土台に次の PR を積むやり方で、依存する変更を小さい単位でレビューしてもらいやすい
- **GUI manifest parser** を renderer 本体から分けると、HTML 以外の renderer を将来追加するときも JSON 読み取り責務を共有しやすい
- **snapshot test** は HTML のように文字列差分で壊れやすい成果物に向いていて、renderer の見た目の骨格をまとめて固定できる
